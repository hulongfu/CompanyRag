# 设计哲学 v8 — CompanyRag 企业知识库 RAG 系统

## 核心原则

### 1. 模块边界清晰，依赖单向
- 各模块职责单一，通过 Maven 模块依赖关系体现架构分层
- 依赖方向：common → tenant/document → rag → agent → web → bootstrap
- 禁止循环依赖和跨层越级调用

### 2. 多租户安全是首要约束
- Schema 物理隔离 + 行级安全策略双重保障
- 所有数据访问必须经过租户上下文过滤
- 租户 ID 通过请求头 `X-Tenant-Id` 传递

### 3. 可观测性是默认能力
- 核心业务流程必须埋点：请求数、延迟、召回率、Token 消耗
- 熔断、限流、超时等工程保障必须可配置、可监控
- 日志分级清晰：ERROR 系统异常、WARN 业务异常、INFO 流程信息、DEBUG 调试细节

### 4. 防御性编程
- 外部依赖（LLM、数据库、缓存）必须有熔断保护
- 输入校验在 Controller 层完成，业务层信任校验后的数据
- 全局异常处理器兜底，不允许未捕获异常泄漏到前端

### 5. AI 集成抽象
- 通过 Spring AI 统一抽象对接 AI 模型，不直接依赖特定厂商 SDK
- 模型配置（model/temperature/topK）外部化，支持运行时切换
- 向量维度、距离算法、索引类型等基础设施配置统一管理

## 架构约束

- 所有 REST API 返回统一格式 `R<T>`（code + message + data）
- 分页查询统一使用 MyBatis-Plus Page 对象
- 文件上传限制 50MB，超过返回友好错误提示
- 敏感信息（API Key、密码）通过环境变量注入，禁止硬编码
