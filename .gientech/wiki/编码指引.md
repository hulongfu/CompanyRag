## 概述

本文档定义了 CompanyRag（企业知识库 RAG 系统）项目的编码规范和最佳实践，旨在确保代码质量、可维护性和团队协作效率。所有开发人员必须严格遵循本规范，通过代码审查和自动化工具确保规范的执行。

### 技术栈概况

| 组件 | 技术选型 |
|------|----------|
| 框架 | Spring Boot 3.4.4 + Spring AI 1.0.4 |
| 语言 | Java 17 |
| 构建工具 | Maven 3.6+ |
| ORM | MyBatis-Plus 3.5.9 |
| 数据库 | PostgreSQL 16 + PGVector |
| 缓存 | Redis (Redisson 3.40.2) |
| AI 模型 | 通义千问 qwen-max + text-embedding-v3 |
| 文档解析 | Apache Tika 3.1.0 |
| 熔断限流 | Resilience4j 2.2.0 |
| 可观测性 | Micrometer + Prometheus + Grafana |

> 来源：[README.md](../../README.md)、[pom.xml](../../pom.xml)

### 核心原则
- **可读性优先**: 代码应该易于理解和维护
- **一致性**: 保持代码风格和结构的一致性
- **简洁性**: 避免过度复杂的设计和实现
- **安全性**: 遵循安全编码最佳实践，敏感信息通过环境变量管理
- **性能**: 在保证可读性的前提下优化性能

## 代码风格规范

### 基本格式规范

#### 缩进和空格
- 使用 4 个空格进行缩进（参考 [CompanyRagApplication.java](../../company-rag-bootstrap/src/main/java/com/company/rag/bootstrap/CompanyRagApplication.java) 的实际代码风格）
- 行尾不允许有空格
- 文件末尾保留一个空行
- 每行代码长度不超过 120 个字符

#### 大括号规范

```java
// 推荐：大括号不换行（K&R 风格）
public class ExampleClass {
    public void method() {
        if (condition) {
            // 代码逻辑
        }
    }
}
```

> 来源：项目中所有源码文件（如 [CompanyRagApplication.java](../../company-rag-bootstrap/src/main/java/com/company/rag/bootstrap/CompanyRagApplication.java)、[GlobalExceptionHandler.java](../../company-rag-common/src/main/java/com/company/rag/common/exception/GlobalExceptionHandler.java)）均采用大括号不换行风格。

#### 空行规范
- 类成员之间使用一个空行分隔
- 方法之间使用一个空行分隔
- 逻辑块之间使用空行提高可读性
- 导入语句分组，组间使用空行分隔

### 导入语句规范

```java
// 1. Java 标准库
import java.util.List;
import java.util.stream.Collectors;

// 2. 第三方库
import lombok.extern.slf4j.Slf4j;
```

## 命名规范

### 类命名规范

#### 类名
- 使用 PascalCase（首字母大写的驼峰命名）
- 类名应该是名词，具有描述性
- 避免缩写，除非是广泛认知的缩写

```java
// 推荐
public class CompanyRagApplication { }
public class GlobalExceptionHandler { }
public class BizException { }

// 不推荐
public class app { }
public class ExcHandler { }
```

> 来源：[CompanyRagApplication.java](../../company-rag-bootstrap/src/main/java/com/company/rag/bootstrap/CompanyRagApplication.java)、[GlobalExceptionHandler.java](../../company-rag-common/src/main/java/com/company/rag/common/exception/GlobalExceptionHandler.java)

#### 接口命名
- 接口名使用描述性名词
- 接口实现类可以添加 "Impl" 后缀

```java
// 推荐
public interface TenantService { }
public class TenantServiceImpl implements TenantService { }
```

### 方法命名规范

#### 方法名
- 使用 camelCase（首字母小写的驼峰命名）
- 方法名应该是动词或动词短语
- 布尔方法使用 is、has、can 等前缀

```java
// 推荐
public void createUser(User user) { }
public List<User> findUsersByRole(String role) { }
public boolean isValidEmail(String email) { }
public boolean hasPermission(String permission) { }
```

> 来源：[CompanyRagApplication.java](../../company-rag-bootstrap/src/main/java/com/company/rag/bootstrap/CompanyRagApplication.java) 中的 `tenantSchemaInitializer`、`getAllTenants`、`createTenantSchema` 等方法命名。

### 变量命名规范

#### 变量名
- 使用 camelCase
- 变量名应该具有描述性
- 避免使用单字母变量名（除了循环变量和 lambda 参数）

```java
// 推荐
String userName = "john";
List<Order> pendingOrders = new ArrayList<>();
int initializedCount = 0;

// 不推荐
String u = "john";
List<Order> list = new ArrayList<>();
int n = 3;
```

> 来源：[CompanyRagApplication.java](../../company-rag-bootstrap/src/main/java/com/company/rag/bootstrap/CompanyRagApplication.java) 中的 `initializedCount`、`tenantName`、`tenantCode` 等变量命名。

#### 常量命名
- 使用 UPPER_SNAKE_CASE
- 常量名应该具有描述性

```java
// 推荐
public static final String HEADER_TENANT_ID = "X-Tenant-Id";
public static final int VECTOR_DIMENSION = 1024;
public static final int DEFAULT_TOP_K = 10;
public static final String CACHE_NAMESPACE = "company:rag:";
```

> 来源：[RagConstant.java](../../company-rag-common/src/main/java/com/company/rag/common/constant/RagConstant.java)

## 代码结构规范

### 类结构规范

#### 类成员顺序

```java
public class ExampleClass {
    // 1. 常量
    private static final String CONSTANT = "value";

    // 2. 静态变量
    private static int staticVariable;

    // 3. 实例变量
    private String instanceVariable;

    // 4. 构造函数
    public ExampleClass() { }

    // 5. 静态方法
    public static void staticMethod() { }

    // 6. 实例方法
    public void instanceMethod() { }

    // 7. 私有方法
    private void privateMethod() { }
}
```

### 方法结构规范

#### 方法长度
- 单个方法不超过 50 行
- 如果方法过长，考虑拆分为多个小方法
- 每个方法只做一件事

#### 参数规范
- 方法参数不超过 5 个
- 如果参数过多，考虑使用参数对象

```java
// 推荐：使用参数对象
public void createUser(UserCreateRequest request) {
    // 实现逻辑
}
```

## 异常处理规范

### 异常分类

#### 业务异常 vs 系统异常
- **业务异常**: 继承 `BizException`（运行时异常），包含业务错误码和消息
- **系统异常**: 使用 RuntimeException/Exception，由全局处理器统一兜底

```java
// 业务异常：可预期的业务错误
public class BizException extends RuntimeException {
    private final int code;

    public BizException(int code, String message) {
        super(message);
        this.code = code;
    }

    public static BizException unauthorized(String message) {
        return new BizException(401, message);
    }

    public static BizException notFound(String message) {
        return new BizException(404, message);
    }
}
```

> 来源：[BizException.java](../../company-rag-common/src/main/java/com/company/rag/common/exception/BizException.java)

### 全局异常处理

使用 `@RestControllerAdvice` 统一处理所有 Controller 抛出的异常，返回统一响应格式：

```java
@Slf4j
@RestControllerAdvice
public class GlobalExceptionHandler {

    // 业务异常处理
    @ExceptionHandler(BizException.class)
    public R<Void> handleBizException(BizException e, HttpServletRequest request) {
        log.warn("业务异常: code={} | path={} | msg={}", 
                e.getCode(), request.getRequestURI(), e.getMessage());
        return R.fail(e.getCode(), e.getMessage());
    }

    // 参数校验异常
    @ExceptionHandler(MethodArgumentNotValidException.class)
    @ResponseStatus(HttpStatus.BAD_REQUEST)
    public R<Void> handleValidationException(MethodArgumentNotValidException e) {
        // 处理参数校验失败
    }

    // 运行时异常兜底
    @ExceptionHandler(RuntimeException.class)
    @ResponseStatus(HttpStatus.INTERNAL_SERVER_ERROR)
    public R<Void> handleRuntimeException(RuntimeException e, HttpServletRequest request) {
        log.error("运行时异常: path={} | msg={}", request.getRequestURI(), e.getMessage(), e);
        return R.fail(500, "系统繁忙，请稍后重试");
    }
}
```

> 来源：[GlobalExceptionHandler.java](../../company-rag-common/src/main/java/com/company/rag/common/exception/GlobalExceptionHandler.java)、[R.java](../../company-rag-common/src/main/java/com/company/rag/common/model/R.java)

### 异常处理最佳实践

#### 具体异常处理
```java
// 推荐：具体异常处理，避免捕获通用 Exception
try {
    userService.createUser(user);
} catch (BizException e) {
    log.warn("业务异常: {}", e.getMessage());
    return R.fail(e.getCode(), e.getMessage());
}
```

#### 日志分级
- 业务异常（BizException）使用 `log.warn` 记录
- 系统异常（RuntimeException/Exception）使用 `log.error` 记录，并打印堆栈

> 来源：[GlobalExceptionHandler.java](../../company-rag-common/src/main/java/com/company/rag/common/exception/GlobalExceptionHandler.java)

## 日志规范

### 日志框架

项目使用 SLF4J + Logback 作为日志门面，通过 Lombok 的 `@Slf4j` 注解注入 Logger 实例。

### 日志级别使用

| 级别 | 使用场景 | 示例 |
|------|----------|------|
| **ERROR** | 系统异常，需要立即处理 | 运行时异常、数据库连接失败 |
| **WARN** | 业务异常或警告信息 | 参数校验失败、业务异常 |
| **INFO** | 重要的业务流程信息 | 启动完成、Schema 初始化成功 |
| **DEBUG** | 调试信息，开发环境使用 | 方法调用细节 |

#### 日志级别配置

项目日志级别在 [application.yml](../../company-rag-bootstrap/src/main/resources/application.yml) 中配置：

```yaml
logging:
  level:
    com.company.rag: DEBUG
    org.springframework.ai: INFO
```

> 来源：[application.yml](../../company-rag-bootstrap/src/main/resources/application.yml)

### 日志消息格式

```java
// 推荐：使用占位符 {} 进行结构化日志
log.info("开始检查并初始化租户 Schema...");
log.info("检测到未初始化 Schema 的租户：{} ({})", 
    tenant.getTenantName(), tenant.getTenantCode());
log.info("本次启动共初始化 {} 个租户的 Schema", initializedCountapse);

log.warn("业务异常: code={} | path={} | msg={}", 
    e.getCode(), request.getRequestURI(), e.getMessage());

// 推荐：错误日志包含堆栈信息
```

> 来源：[CompanyRagApplication.java](../../company-rag-bootstrap/src/main/java/com/company/rag/bootstrap/CompanyRagApplication.java)、[GlobalExceptionHandler.java](../../company-rag-common/src/main/java/com/company/rag/common/exception/GlobalExceptionHandler.java)

### 日志安全
- **禁止记录敏感信息**：日志中不得记录密码、API Key 等敏感信息
- **用户标识脱敏**：记录用户相关操作时，使用 userId 而非用户名

## 安全编码规范

### 敏感信息管理

#### 环境变量管理
敏感信息（API Key、数据库密码、Redis 密码等）通过环境变量或 `.env` 文件管理，禁止硬编码在代码中：

```yaml
spring:
  ai:
    openai:
      api-key: ${DASHSCOPE_API_KEY}
  datasource:
    password: ${POSTGRES_PASSWORD}
```

> 来源：[application.yml](../../company-rag-bootstrap/src/main/resources/application.yml)、[.env](../../.env)

#### 配置文件脱敏
配置文件中包含的敏感信息应在提交前脱敏，或在文档引用时使用 `***` 替代。

### 输入验证

#### 参数校验
使用 Bean Validation 注解进行参数校验：

```java
public class UserCreateRequest {
    @NotBlank(message = "用户名不能为空")
    @Size(min = 3, max = 20, message = "用户名长度必须在3-20个字符之间")
    private String username;

    @Email(message = "邮箱格式不正确")
    private String email;
}
```

#### 全局校验异常处理
参数校验失败由 [GlobalExceptionHandler.java](../../company-rag-common/src/main/java/com/company/rag/common/exception/GlobalExceptionHandler.java) 统一处理，返回标准错误响应。

### 文件上传安全

项目在 [application.yml](../../company-rag-bootstrap/src/main/resources/application.yml) 中限制了文件上传大小：

```yaml
spring:
  servlet:
    multipart:
      max-file-size: 50MB
      max-request-size: 100MB
```

文件大小超限时，由 `GlobalExceptionHandler` 统一返回友好的错误提示，避免信息泄露。

### 熔断与限流

项目使用 Resilience4j 保护外部服务调用（如 LLM API 调用），防止级联故障：

```yaml
resilience4j:
  circuitbreaker:
    configs:
      default:
        sliding-window-size: 10
        minimum-number-of-calls: 5
        failure-rate-threshold: 50
        wait-duration-in-open-state: 30s
  ratelimiter:
    configs:
      default:
        limit-for-period: 10
        limit-refresh-period: 1s
        timeout-duration: 500ms
```

> 来源：[application.yml](../../company-rag-bootstrap/src/main/resources/application.yml)
