# 多租户架构

## 概述

CompanyRag 采用双重租户隔离策略：PostgreSQL Schema 物理隔离 + 行级安全（RLS）策略，配合 MyBatis-Plus 租户拦截器实现三层防御。

## 租户隔离机制

### 三层防御模型

```
第一层：请求拦截器 (TenantInterceptor)
  └─ 从 X-Tenant-Id 请求头解析租户信息，注入 ThreadLocal

第二层：Schema 隔离 (TenantContextHelper)
  └─ 动态切换 PostgreSQL search_path，路由到租户独立 Schema

第三层：行级安全 (RLS + MyBatis-Plus 拦截器)
  └─ RLS 策略限制行级访问 + MyBatis-Plus 自动追加 tenant_id 条件
```

### 1. ThreadLocal 上下文 (`TenantContext`)

`TenantContext` 使用 `ThreadLocal` 存储当前请求的租户信息：

- `tenantId` — 租户 ID
- `tenantCode` — 租户编码
- `userId` — 当前用户 ID
- `schema` — 租户 Schema 名称

### 2. 请求拦截器 (`TenantInterceptor`)

请求进入时：
1. 从 `X-Tenant-Id` 请求头（或 `tenantId` 参数）解析租户 ID
2. 查询 `sys_tenant` 获取 Schema 名称
3. 调用 `TenantContextHelper.setTenantContext()` 切换 Schema + 设置 RLS session 变量
4. 设置 `TenantContext` ThreadLocal

请求结束时：
1. 调用 `TenantContext.clear()` 清理 ThreadLocal
2. 调用 `TenantContextHelper.resetSchema()` 恢复 `search_path TO public`

### 3. Schema 动态切换 (`TenantContextHelper`)

- `setTenantContext(tenantId, schemaName)` — 同时设置 RLS session 变量和 Schema
- `setCurrentTenant(tenantId)` — 执行 `SET app.tenant_id = {tenantId}`
- `setSchema(schemaName)` — 执行 `SET search_path TO {schemaName}, public`
- `resetSchema()` — 执行 `SET search_path TO public`

**安全注意**：Schema 名称经过正则校验 `^[a-zA-Z_][a-zA-Z0-9_]*$`，防止 SQL 注入。

### 4. MyBatis-Plus 租户拦截器 (`TenantLineInnerInterceptor`)

自动为所有 SQL 追加 `tenant_id = ?` 条件：
- 从 `TenantContext.getTenantId()` 获取当前租户 ID
- 租户字段为 `tenant_id`
- `sys_tenant` 表被排除（忽略租户隔离）

### 5. PostgreSQL 行级安全 (RLS)

每个租户 Schema 中启用 RLS，策略为：
```sql
CREATE POLICY tenant_isolation_document ON rag_document
    USING (tenant_id = current_tenant_id() OR current_user = 'postgres');
```

## 租户创建流程

`createTenantWithSchema(tenant)` 完整流程：

1. 校验 `tenantCode` 唯一性
2. 插入 `sys_tenant` 记录
3. 创建独立 Schema：`CREATE SCHEMA IF NOT EXISTS tenant_{code}`
4. 在 Schema 中创建业务表：`rag_document`、`doc_chunk`、`vector_store`、`rag_session`
5. 创建索引：B-Tree、GIN（全文检索）、HNSW（向量）
6. 启用 RLS 并创建租户隔离策略
7. 创建默认管理员用户（admin/admin123）

## 数据模型

### sys_tenant

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键 |
| tenant_code | VARCHAR(64) | 租户编码（唯一） |
| tenant_name | VARCHAR(128) | 租户名称 |
| schema_name | VARCHAR(128) | 对应 Schema 名称 |
| status | INTEGER | 状态（1 正常 / 0 禁用） |
| create_time | TIMESTAMP | 创建时间 |

### sys_user

| 字段 | 类型 | 说明 |
|------|------|------|
| id | BIGSERIAL | 主键 |
| tenant_id | BIGINT | 所属租户 |
| username | VARCHAR(64) | 用户名 |
| password | VARCHAR(128) | BCrypt 加密密码 |
| display_name | VARCHAR(64) | 显示名称 |
| role | VARCHAR(32) | 角色（admin/user/viewer） |
| status | INTEGER | 状态 |

## 请求头约定

| 请求头 | 说明 | 示例 |
|--------|------|------|
| `X-Tenant-Id` | 租户 ID | `1` |
| `X-User-Id` | 用户 ID | `100` |
