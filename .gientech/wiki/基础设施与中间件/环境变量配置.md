## 简介

CompanyRag 项目采用**环境变量注入**方式管理敏感配置信息，包括 API 密钥、数据库连接、Redis 连接等。通过 `.env` 文件与 Spring Boot 配置文件的配合，实现配置与代码分离，支持多环境部署。

**核心特性**：
- 敏感信息（API Key、密码）通过环境变量注入，不硬编码
- 支持开发（dev）和生产（prod）多环境配置
- 所有配置项提供默认值，便于本地快速启动
- 符合 12-Factor App 配置原则

## 环境变量清单

### DashScope API 配置

| 变量名 | 说明 | 示例值 | 必填 |
| ------| ------| --------| ------|
| `DASHSCOPE_API_KEY` | 通义千问 Chat 模型 API 密钥 | `sk-***` | 是 |
| `SILICONFLOW_API_KEY` | 硅基流动 Embedding 模型 API 密钥 | `sk-***` | 是 |

### 数据库配置

| 变量名 | 说明 | 默认值 | 必填 |
| ------| ------| --------| ------|
| `POSTGRES_HOST` | PostgreSQL 主机地址 | `localhost` | 否 |
| `POSTGRES_PORT` | PostgreSQL 端口 | `5433` | 否 |
| `POSTGRES_DB` | 数据库名称 | `company_rag` | 否 |
| `POSTGRES_USER` | 数据库用户名 | `postgres` | 否 |
| `POSTGRES_PASSWORD` | 数据库密码 | 无默认值 | 是 |

### Redis 配置

| 变量名 | 说明 | 默认值 | 必填 |
| ------| ------| --------| ------|
| `REDIS_HOST` | Redis 主机地址 | `localhost` | 否 |
| `REDIS_PORT` | Redis 端口 | `6379` | 否 |
| `REDIS_PASSWORD` | Redis 密码 | 无默认值 | 否 |

### 服务配置

| 变量名 | 说明 | 默认值 | 必填 |
| ------| ------| --------| ------|
| `SERVER_PORT` | 服务启动端口 | `8080` | 否 |
| `SPRING_PROFILES_ACTIVE` | Spring 激活的 profile | `dev` | 否 |

## 配置注入方式

### Spring Boot 环境变量绑定

项目使用 `${ENV_VAR:default}` 语法在 `application.yml` 中注入环境变量：

```yaml
server:
  port: ${SERVER_PORT:8080}

spring:
  datasource:
    url: jdbc:postgresql://${POSTGRES_HOST:localhost}:${POSTGRES_PORT:5433}/${POSTGRES_DB:company_rag}
    username: ${POSTGRES_USER:postgres}
    password: ${POSTGRES_PASSWORD:}
  
  data:
    redis:
      host: ${REDIS_HOST:localhost}
      port: ${REDIS_PORT:6379}
      password: ${REDIS_PASSWORD:}
```

**来源**：[application.yml](../../../company-rag-bootstrap/src/main/resources/application.yml)(L1-L45)

### AI 模型配置

Chat 模型和 Embedding 模型分别使用不同的 API Key：

```yaml
spring:
  ai:
    openai:
      # Chat 模型（通义千问）
      api-key: ${DASHSCOPE_API_KEY:default_dashscope_api_key}
      base-url: https://dashscope.aliyuncs.com/compatible-mode
      
      # Embedding 模型（硅基流动）
      embedding:
        api-key: ${SILICONFLOW_API_KEY:default_siliconflow_api_key}
        base-url: https://api.siliconflow.cn
```

**来源**：[application-dev.yml](../../../company-rag-bootstrap/src/main/resources/application-dev.yml)(L4-L19)

## 多环境配置

### 开发环境（dev）

开发环境配置特点：
- 数据库和 Redis 使用本地地址
- 日志级别为 DEBUG
- 使用默认 API Key（便于测试）

**来源**：[application-dev.yml](../../../company-rag-bootstrap/src/main/resources/application-dev.yml)

### 生产环境（prod）

生产环境配置特点：
- 数据库和 Redis 使用 Docker 服务名（`postgres`、`redis`）
- 日志级别为 INFO
- 必须配置真实的 API Key

```yaml
spring:
  datasource:
    url: jdbc:postgresql://postgres:5433/company_rag?currentSchema=public
  ai:
    openai:
      api-key: ${DASHSCOPE_API_KEY:you_dashscope_api_key}
  data:
    redis:
      host: redis
      port: 6379
```

**来源**：[application-prod.yml](../../../company-rag-bootstrap/src/main/resources/application-prod.yml)(L1-L28)

## 使用示例

### 本地开发启动

1. **复制环境变量模板**：
```bash
cp .env.example .env
```

2. **编辑 .env 文件**，填入真实配置：
```bash
# DashScope API 配置
DASHSCOPE_API_KEY=sk-your-actual-api-key

# 数据库配置
POSTGRES_PASSWORD=your_database_password

# Redis 配置
REDIS_PASSWORD=your_redis_password
```

3. **启动应用**：
```bash
mvn spring-boot:run -pl company-rag-bootstrap
```

### Docker Compose 部署

在生产环境中，环境变量通过 Docker Compose 注入：

```yaml
services:
  company-rag:
    image: company-rag:latest
    env_file:
      - .env
    ports:
      - "8080:8080"
```

**注意**：实际部署命令需参考项目部署文档，此处仅说明环境变量注入方式。

## 安全注意事项

### 敏感信息保护

1. **禁止提交 .env 文件到 Git**：
   - `.env` 已加入 `.gitignore`
   - 只提交 `.env.example` 模板文件

2. **API Key 脱敏**：
   - 实际配置文件中 API Key 应使用 `***` 脱敏
   - 日志中禁止打印完整 API Key 和密码

3. **生产环境隔离**：
   - 生产环境使用独立的 API Key 和数据库密码
   - 禁止开发环境和生产环境共用凭据

### 配置文件安全

```yaml
# 错误示例：硬编码 API Key
api-key: sk-e1837008725e4748aebf485575230cc6

# 正确示例：使用环境变量
api-key: ${DASHSCOPE_API_KEY:default_value}
```

## 故障排除

### 常见问题及解决方案

#### 1. 启动时报"Missing API Key"错误

**问题现象**：应用启动失败，提示 DashScope API Key 未配置

**排查步骤**：
1. 检查 `.env` 文件是否存在
2. 确认 `DASHSCOPE_API_KEY` 变量已定义
3. 验证 API Key 格式是否正确（以 `sk-` 开头）

**解决方案**：
```bash
# 复制模板并配置
cp .env.example .env
# 编辑 .env，填入真实的 API Key
```

#### 2. 数据库连接失败

**问题现象**：启动时提示 PostgreSQL 连接超时

**排查步骤**：
1. 检查 `POSTGRES_HOST` 和 `POSTGRES_PORT` 是否正确
2. 确认 PostgreSQL 服务是否运行
3. 验证 `POSTGRES_PASSWORD` 是否正确

**解决方案**：
```bash
# 本地开发环境
POSTGRES_HOST=localhost
POSTGRES_PORT=5433

# Docker 环境
POSTGRES_HOST=postgres
POSTGRES_PORT=5433
```

#### 3. Redis 连接被拒绝

**问题现象**：Redis 连接失败，提示认证错误

**排查步骤**：
1. 检查 `REDIS_HOST` 和 `REDIS_PORT` 配置
2. 确认 Redis 是否需要密码
3. 验证 `REDIS_PASSWORD` 是否正确

**解决方案**：
```yaml
# 本地无密码 Redis
REDIS_PASSWORD=

# 生产环境 Redis
REDIS_PASSWORD=your_secure_password
```

### 启用调试日志

在 `application-dev.yml` 中设置日志级别：

```yaml
logging:
  level:
    com.company.rag: DEBUG
```

## 总结

CompanyRag 项目的环境变量配置体系具有以下特点：

1. **配置与代码分离**：通过 `.env` 文件管理敏感信息，符合 12-Factor App 原则
2. **多环境支持**：dev/prod 两套配置，通过 `SPRING_PROFILES_ACTIVE` 切换
3. **安全优先**：API Key 和密码通过环境变量注入，禁止硬编码
4. **易于部署**：支持本地开发和 Docker Compose 部署两种模式

**核心配置文件**：
- `.env.example` - 环境变量模板
- `application.yml` - 主配置文件
- `application-dev.yml` - 开发环境配置
- `application-prod.yml` - 生产环境配置
