# 部署与运维

## 部署方式

### Docker Compose 一键部署（推荐）

```bash
# 配置环境变量
export DASHSCOPE_API_KEY=sk-your-api-key
export SILICONFLOW_API_KEY=sk-your-siliconflow-key

# 启动所有服务
docker compose up -d

# 查看日志
docker compose logs -f app
```

服务列表：

| 服务 | 端口 | 说明 |
|------|------|------|
| postgres | 5432 | PostgreSQL 16 + PGVector |
| redis | 6379 | Redis 缓存 |
| app | 8080 | Spring Boot 应用 |
| prometheus | 9090 | Prometheus 监控 |
| grafana | 3000 | Grafana 可视化（admin/admin） |

### 独立编译部署

```bash
# 1. 编译
mvn clean package -DskipTests

# 2. 启动
java -jar company-rag-bootstrap/target/company-rag-bootstrap-1.0.0-SNAPSHOT.jar

# 3. 访问 http://localhost:8080
```

## 基础设施

### PostgreSQL + PGVector

- 镜像：`pgvector/pgvector:pg16`
- 数据库：`company_rag`
- 用户：`postgres` / 密码：`postgres`
- 初始化脚本：`sql/init.sql` 自动执行

### Redis

- 镜像：`redis:7-alpine`
- 端口：6379
- 用途：RAG 缓存、限流、会话存储

### Prometheus

- 镜像：`prom/prometheus:latest`
- 端口：9090
- 配置：`prometheus.yml`
- 抓取端点：`/actuator/prometheus`

### Grafana

- 镜像：`grafana/grafana:latest`
- 端口：3000
- 默认账号：`admin` / `admin`
- 数据源：Prometheus（需手动配置）

## 健康检查端点

| 端点 | 说明 |
|------|------|
| GET /actuator/health | 应用健康状态（UP/DOWN） |
| GET /actuator/info | 应用信息（版本、环境） |
| GET /actuator/prometheus | Prometheus 指标 |
| GET /actuator/metrics | Micrometer 指标 |

## Prometheus 指标

### 请求指标
- `rag_requests_total` — RAG 请求总数
- `rag_request_total_seconds` — 请求总耗时
- `rag_retrieval_duration_seconds` — 检索阶段耗时
- `rag_rerank_duration_seconds` — Rerank 阶段耗时
- `rag_llm_duration_seconds` — LLM 调用耗时

### 缓存指标
- `rag_cache_hits_total` — 缓存命中次数
- `rag_cache_misses_total` — 缓存未命中次数

### Token 指标
- `rag_tokens_input_total` — 输入 Token 数
- `rag_tokens_output_total` — 输出 Token 数
- `rag_tokens_total_total` — 总 Token 数

### 限流指标
- `rag_ratelimit_hits_total` — 限流触发次数

## Grafana 面板配置

1. 添加数据源：Prometheus（URL: `http://prometheus:9090`）
2. 导入 Dashboard 或手动创建面板
3. 推荐面板：
   - RAG 请求延迟分布
   - 缓存命中率
   - Token 消耗趋势
   - LLM 调用成功率

## 常见问题

### 1. 应用启动失败
检查环境变量 `DASHSCOPE_API_KEY` 和 `SILICONFLOW_API_KEY` 是否正确配置。

### 2. 数据库连接失败
确认 PostgreSQL 容器已启动并执行 `docker compose logs postgres` 查看日志。

### 3. 缓存未命中率高
检查 Redis 连接配置，确认网络通畅。

### 4. LLM 调用超时
增加 `resilience4j.timelimiter.timeout-duration` 配置（默认 30 秒）。
