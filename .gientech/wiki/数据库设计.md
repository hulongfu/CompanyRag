# 数据库设计

## 概述

数据库设计遵循以下原则：
1. **public schema** 只存放系统级表（租户、用户）
2. **租户业务表** 独立存放在 `tenant_{code}` schema
3. 通过 Schema 实现物理数据隔离

## 扩展

```sql
CREATE EXTENSION IF NOT EXISTS vector;   -- PGVector
CREATE EXTENSION IF NOT EXISTS pg_trgm;  -- 全文检索
```

## 系统表 (public schema)

### sys_tenant — 租户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGSERIAL | PRIMARY KEY | 主键 |
| tenant_code | VARCHAR(64) | NOT NULL UNIQUE | 租户编码 |
| tenant_name | VARCHAR(128) | NOT NULL | 租户名称 |
| schema_name | VARCHAR(64) | | 独立 Schema 名称 |
| status | INTEGER | DEFAULT 1 | 0-禁用 1-启用 |
| contact_name | VARCHAR(64) | | 联系人 |
| contact_phone | VARCHAR(20) | | 联系电话 |
| create_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

### sys_user — 用户表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGSERIAL | PRIMARY KEY | 主键 |
| tenant_id | BIGINT | NOT NULL FK → sys_tenant | 所属租户 |
| username | VARCHAR(64) | NOT NULL | 用户名 |
| password | VARCHAR(256) | NOT NULL | BCrypt 加密密码 |
| display_name | VARCHAR(128) | | 显示名称 |
| email | VARCHAR(128) | | 邮箱 |
| role | VARCHAR(32) | DEFAULT 'user' | admin / user / viewer |
| status | INTEGER | DEFAULT 1 | 状态 |
| create_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

唯一约束：`(tenant_id, username)`

## 租户业务表 (tenant_{code} schema)

以下表通过 `TenantServiceImpl.createTenantSchema()` 动态创建到每个租户的独立 Schema 中。

### rag_document — 文档表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGSERIAL | PRIMARY KEY | 主键 |
| tenant_id | BIGINT | NOT NULL | 租户 ID |
| file_name | VARCHAR(256) | NOT NULL | 文件名 |
| file_type | VARCHAR(32) | | 文件类型 |
| file_size | BIGINT | | 文件大小 |
| file_path | VARCHAR(512) | | 文件路径 |
| title | VARCHAR(256) | | 文档标题 |
| chunk_count | INTEGER | DEFAULT 0 | 切分块数量 |
| status | INTEGER | DEFAULT 0 | -1失败 0待处理 1已切分 2已向量化 |
| error_msg | TEXT | | 错误信息 |
| create_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| update_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 更新时间 |

### doc_chunk — 文档切分块表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGSERIAL | PRIMARY KEY | 主键 |
| document_id | BIGINT | NOT NULL FK → rag_document (CASCADE) | 所属文档 |
| tenant_id | BIGINT | NOT NULL | 租户 ID |
| chunk_index | INTEGER | NOT NULL | 块序号 |
| content | TEXT | NOT NULL | 文本内容 |
| token_count | INTEGER | DEFAULT 0 | Token 估算数量 |
| split_strategy | VARCHAR(32) | | 切分策略 |
| create_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### vector_store — 向量存储表 (PGVector)

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | UUID | PRIMARY KEY | 向量 ID |
| content | TEXT | | 文本内容 |
| metadata | JSONB | | 元数据（documentId、tenantId、documentName 等） |
| embedding | vector(1024) | | 1024 维向量 |

索引：`USING hnsw (embedding vector_cosine_ops) WITH (m=16, ef_construction=64)`

### rag_session — 对话历史表

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | BIGSERIAL | PRIMARY KEY | 主键 |
| session_id | VARCHAR(128) | NOT NULL | 会话 ID |
| tenant_id | BIGINT | NOT NULL | 租户 ID |
| user_id | BIGINT | | 用户 ID |
| query | TEXT | NOT NULL | 用户查询 |
| answer | TEXT | | 模型回答 |
| context | TEXT | | 检索上下文 |
| tokens_input | INTEGER | DEFAULT 0 | 输入 Token 数 |
| tokens_output | INTEGER | DEFAULT 0 | 输出 Token 数 |
| latency_ms | INTEGER | DEFAULT 0 | 延迟(毫秒) |
| create_time | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

## 索引

| 索引名 | 表 | 类型 | 字段 |
|--------|------|------|------|
| idx_doc_tenant | rag_document | B-Tree | (tenant_id) |
| idx_chunk_document | doc_chunk | B-Tree | (document_id) |
| idx_chunk_document_tenant | doc_chunk | B-Tree | (tenant_id, document_id) |
| idx_session_tenant | rag_session | B-Tree | (tenant_id, session_id) |
| idx_chunk_content_trgm | doc_chunk | GIN | content gin_trgm_ops |
| idx_document_title_trgm | rag_document | GIN | title gin_trgm_ops |
| idx_vector_store_embedding | vector_store | HNSW | embedding vector_cosine_ops |

## 行级安全 (RLS)

```sql
ALTER TABLE rag_document ENABLE ROW LEVEL SECURITY;
ALTER TABLE doc_chunk ENABLE ROW LEVEL SECURITY;
ALTER TABLE rag_session ENABLE ROW LEVEL SECURITY;

CREATE POLICY tenant_isolation_document ON rag_document
    USING (tenant_id = current_tenant_id() OR current_user = 'postgres');
-- doc_chunk 和 rag_session 策略类似
```

## 辅助函数

```sql
-- 获取当前租户 ID
CREATE OR REPLACE FUNCTION current_tenant_id() RETURNS BIGINT
-- 设置当前租户 ID
CREATE OR REPLACE FUNCTION set_tenant_id(p_tenant_id BIGINT) RETURNS VOID
```

## PGVector 配置

- **向量维度**：1024（OpenAI 兼容 Embedding 模型，默认硅基流动 text-embedding）
- **索引类型**：HNSW
- **距离算法**：COSINE_DISTANCE
- **HNSW 参数**：m=16, ef_construction=64

## 默认数据

- 默认租户：`default` / `tenant_default` / 状态=启用
- 默认管理员：admin / admin123（BCrypt 加密）
