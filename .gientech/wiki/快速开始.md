## 简介

本快速入门指南旨在帮助新开发者快速搭建 **CompanyRag**（企业知识库RAG系统）项目的本地开发环境。该系统是一个基于 **Spring Boot 3.4 + Spring AI 1.0 + PGVector + 通义千问** 的企业级检索增强生成系统，支持文档解析、智能切分、向量化存储、混合检索、重排序与流式回答等完整RAG链路。

> 项目描述详见 [README.md](../../README.md)。

## 系统要求

在开始之前，请确保您的系统满足以下要求：

- **JDK版本**: JDK 17 或更高版本
- **构建工具**: Maven 3.6+
- **内存**: 至少 4GB RAM
- **磁盘空间**: 至少 2GB 可用空间
- **网络**: 稳定的互联网连接（需访问 DashScope API）
- **操作系统**: Windows / Linux / macOS
- **其他**: Docker & Docker Compose（用于启动数据库和缓存服务）

> JDK版本要求来源于 [pom.xml](../../pom.xml) 中 `<java.version>17</java.version>` 配置。

## 项目克隆

首先，您需要克隆项目到本地开发环境：

```bash
git clone https://github.com/hulongfu/CompanyRag.git
cd CompanyRag
```

## 环境准备

### 1. 安装JDK 17

确保您的系统已安装JDK 17或更高版本：

```bash
java -version
javac -version
```

如果未安装，请从 [Oracle官网](https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html) 或 [Eclipse Temurin](https://adoptium.net/temurin/releases/?version=17) 下载并安装。

### 2. 安装Maven

项目使用Maven作为构建工具。您可以使用以下方式安装：

#### 方法一：通过SDKMAN!安装（推荐）
```bash
curl -s "https://get.sdkman.io" | bash
source "$HOME/.sdkman/bin/sdkman-init.sh"
sdk install maven 3.9.9
```

#### 方法二：手动安装
从 [Maven官网](https://maven.apache.org/download.cgi) 下载并配置环境变量。

### 3. 验证Maven安装
```bash
mvn -v
```

## 数据库和缓存服务启动

项目依赖 **PostgreSQL 16 + PGVector** 数据库和 **Redis 7** 缓存服务。我们使用Docker Compose来简化服务部署。

### 启动Docker容器

在项目根目录下执行以下命令：

```bash
docker compose up -d postgres redis
```

这将启动以下服务：
- **PostgreSQL 16 + PGVector**（端口: 5432）— 业务数据与向量数据存储
- **Redis 7**（端口: 6379）— 缓存、限流与会话管理

### 验证服务状态

检查所有容器是否正常运行：

```bash
docker ps
```

您应该看到 `company-rag-postgres` 和 `company-rag-redis` 两个容器正在运行。

### 初始化数据库

首次启动时，PostgreSQL容器会自动执行 `sql/init.sql` 初始化脚本。您可以查看 [sql/README.md](../../sql/README.md) 了解数据表结构概览，包括以下核心表：

| 表名 | 说明 |
|------|------|
| sys_tenant | 租户信息 |
| sys_user | 用户信息（tenant_id隔离） |
| rag_document | 文档元数据（tenant_id隔离） |
| doc_chunk | 文档切分块（tenant_id隔离） |
| vector_store | 向量存储（PGVector） |
| rag_session | 对话历史（tenant_id隔离） |

## 项目导入到IDE

### IntelliJ IDEA导入

1. 打开IntelliJ IDEA
2. 选择 **File** → **Open...**
3. 浏览到项目目录并选择
4. 选择 **Import project from external model**
5. 选择 **Maven** 并点击 **Next**
6. 配置Maven设置（建议使用默认设置）
7. 点击 **Finish**

IDEA会自动识别多模块Maven项目结构，包含以下模块：
- `company-rag-common` — 公共模块
- `company-rag-tenant` — 多租户模块
- `company-rag-document` — 文档解析与切分模块
- `company-rag-rag` — RAG核心检索模块
- `company-rag-agent` — Agent工具模块
- `company-rag-web` — Web层（Controller + 前端页面）
- `company-rag-bootstrap` — 启动模块

## 构建和运行项目

### 配置环境变量

在运行前，需要配置 DashScope API Key。您可以在 [DashScope](https://dashscope.aliyun.com) 申请：

```bash
# Windows (cmd)
set DASHSCOPE_API_KEY=sk-your-api-key

# Linux/Mac
export DASHSCOPE_API_KEY=sk-your-api-key
```

也可以修改 `.env` 文件中的 `DASHSCOPE_API_KEY` 配置，或直接在 [application.yml](../../company-rag-bootstrap/src/main/resources/application.yml) 中设置。

### 构建项目

在项目根目录下执行：

```bash
mvn clean package -DskipTests
```

### 运行应用程序

有两种方式启动应用：

#### 方式一：直接运行Jar包
```bash
java -jar company-rag-bootstrap/target/company-rag-bootstrap-1.0.0-SNAPSHOT.jar
```

#### 方式二：在IDE中运行
在IDE中找到 `CompanyRagApplication.java` 文件，右键选择 **Run 'CompanyRagApplication'**。

> 入口类位于 [CompanyRagApplication.java](../../company-rag-bootstrap/src/main/java/com/company/rag/bootstrap/CompanyRagApplication.java)。

### 验证应用启动

应用启动后，您应该能看到类似以下的日志输出：

```
Tomcat started on port(s): 8080 (http) with context path ''
Started CompanyRagApplication in XX.XXX seconds
```

应用启动后会自动检查并初始化租户Schema。

### Docker Compose 完整部署

如需完整部署所有服务（含应用本身），可执行：

```bash
export DASHSCOPE_API_KEY=sk-your-api-key
docker compose up -d
```

这将启动5个服务：postgres、redis、app、prometheus、grafana。

## 故障排除

### 常见问题及解决方案

#### 1. 端口冲突
如果8080端口被占用，可以在 `application.yml` 中修改端口配置：

```yaml
server:
  port: 8081
```

#### 2. 数据库连接失败
检查数据库服务是否正常运行：

```bash
docker logs company-rag-postgres
```

确认连接信息与 [application.yml](../../company-rag-bootstrap/src/main/resources/application.yml) 中的 `spring.datasource.url` 配置一致。

#### 3. Redis连接失败
检查Redis服务是否正常运行：

```bash
docker logs company-rag-redis
```

确认Redis密码与 [application.yml](../../company-rag-bootstrap/src/main/resources/application.yml) 中的 `spring.data.redis.password` 配置一致。

#### 4. 构建失败
清理并重新构建：

```bash
mvn clean package -DskipTests --refresh-dependencies
```

#### 5. API Key未配置
如果遇到 `401` 或模型调用失败，请确认已正确配置 `DASHSCOPE_API_KEY` 环境变量。可在 [DashScope](https://dashscope.aliyun.com) 申请API Key。

#### 6. 内存不足
增加JVM堆内存：

```bash
java -Xms512m -Xmx2g -jar company-rag-bootstrap/target/company-rag-bootstrap-1.0.0-SNAPSHOT.jar
