# 不可违反规则 — CompanyRag

## 规则列表

### R1: 禁止循环依赖
任何模块间不允许出现循环依赖。依赖方向必须为：common → tenant/document → rag → agent → web → bootstrap。

### R2: 所有数据访问必须经过租户隔离
任何 SQL 查询、数据写入操作必须携带 `tenant_id` 过滤条件。Schema 级隔离由 MyBatis-Plus 拦截器自动处理，但业务代码也要显式传递租户上下文。

### R3: 外部 LLM 调用必须配置熔断保护
所有通过 Spring AI 发起的 LLM 调用（Chat / Embedding）必须使用 Resilience4j CircuitBreaker 包装，并配置合理的超时时间（默认 30 秒）。

### R4: API 统一响应格式
所有 REST API 必须返回 `R<T>` 格式，禁止直接返回裸对象或其它格式。全局异常处理器负责兜底所有未捕获异常。

### R5: 敏感信息禁止硬编码
数据库密码、API Key、Redis 密码等敏感信息必须通过环境变量注入，禁止在代码或配置文件中硬编码。提交前需脱敏处理。

### R6: 向量维度必须一致
所有向量化操作必须使用相同维度（1024）和距离算法（COSINE_DISTANCE），索引类型统一为 HNSW。向量维度变更需要同步更新数据库索引。

### R7: 跨模块调用禁止越层
Controller 只能调用本模块或下层模块的 Service，禁止跨层调用。例如 web 模块不能直接调用 document 模块的 Mapper。

### R8: 文件上传必须限制大小
文件上传必须限制单文件不超过 50MB，总请求不超过 100MB。超过限制返回友好错误提示，禁止直接抛出系统异常。

### R9: 日志禁止记录敏感信息
任何日志输出中不得包含密码、API Key、Token 等敏感信息。使用占位符 `{}` 进行参数化日志记录。

### R10: 核心操作必须记录可观测性指标
RAG 检索、LLM 调用、文档解析等核心操作必须通过 Micrometer 记录请求数、延迟、错误率等指标。
