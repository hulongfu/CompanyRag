# ✅ 最终修复完成 - 使用OpenAI兼容协议

## 🔧 核心修复

### 问题根因
- **Spring AI 1.0.4 不包含 `spring-ai-dashscope`**
- **Spring AI 1.1.0 与 Spring Boot 3.4.4 不兼容**

### 解决方案
**使用 DashScope 的 OpenAI 兼容协议！**

---

## 📋 修改内容

### 1. POM依赖修改

**修改前**:
```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-dashscope-spring-boot-starter</artifactId>
</dependency>
```

**修改后**:
```xml
<dependency>
    <groupId>org.springframework.ai</groupId>
    <artifactId>spring-ai-openai-spring-boot-starter</artifactId>
</dependency>
```

**修改文件**:
- `company-rag-rag/pom.xml`
- `company-rag-agent/pom.xml`

---

### 2. 配置文件修改

**修改前**:
```yaml
spring:
  ai:
    dashscope:
      api-key: ${DASHSCOPE_API_KEY}
      chat:
        options:
          model: qwen-max
```

**修改后**:
```yaml
spring:
  ai:
    openai:
      api-key: ${DASHSCOPE_API_KEY}
      base-url: https://dashscope.aliyuncs.com/compatible-mode/v1
      chat:
        options:
          model: qwen-max
      embedding:
        options:
          model: text-embedding-v3
```

**关键点**:
- `base-url` 指向 DashScope 的 OpenAI 兼容端点
- 使用相同的 API Key
- 模型名称保持不变

---

### 3. Java代码修改

**修改前**:
```java
import org.springframework.ai.dashscope.DashscopeChatModel;

private final DashscopeChatModel chatModel;
```

**修改后**:
```java
import org.springframework.ai.openai.OpenAiChatModel;

private final OpenAiChatModel chatModel;
```

**修改文件**:
- `RagSearchServiceImpl.java`
- `CrossEncoderReranker.java`
- `RagAgentService.java`

## ⚠️ 可能遇到的问题

### 问题1: 数据库连接失败

**错误**: `Connection refused: localhost:5433`

**解决**:
```bash
# 启动PostgreSQL容器
docker run -d \
  --name company-rag-postgres \
  -e POSTGRES_DB=company_rag \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=difyai123456 \
  -p 5433:5432 \
  pgvector/pgvector:pg16

# 执行初始化脚本
docker exec -i company-rag-postgres psql -U postgres -d company_rag < sql/init.sql
```

---

### 问题2: Redis连接失败

**错误**: `Unable to connect to Redis`

**解决**:
```bash
docker run -d \
  --name company-rag-redis \
  -p 6379:6379 \
  redis:7-alpine
```

---

### 问题3: API Key无效

**错误**: `Invalid API key`

**解决**:
1. 访问 https://dashscope.console.aliyun.com/
2. 获取API Key
3. 设置环境变量:
   ```bash
   # Windows
   set DASHSCOPE_API_KEY=sk-xxxxxxxx
   
   # Linux/Mac
   export DASHSCOPE_API_KEY=sk-xxxxxxxx
   ```

## 📊 技术原理

### 为什么使用OpenAI兼容协议?

**DashScope OpenAI兼容模式**:
- ✅ 完全兼容OpenAI API规范
- ✅ 无需修改代码,只需改配置
- ✅ 支持所有通义千问模型
- ✅ 与Spring AI完美集成

**端点映射**:
```
OpenAI端点:     https://api.openai.com/v1
DashScope端点:  https://dashscope.aliyuncs.com/compatible-mode/v1
```

**模型映射**:
| OpenAI模型 | DashScope模型 |
|-----------|--------------|
| gpt-4 | qwen-max |
| gpt-3.5-turbo | qwen-plus |
| text-embedding-ada-002 | text-embedding-v3 |
